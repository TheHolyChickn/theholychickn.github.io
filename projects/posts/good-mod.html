Good Mod

Category: Minecraft Modding

This page is still a work in progress, not all features stated herein exist yet!

Overview and Features

Good Mod is a Minecraft mod designed for Hypixel Skyblock, a popular Minecraft RPG. The mod is designed to implement a small number of QOL features that are either not offered by other mainstream mods, are implemented poorly by other mainstream mods, or are only done well by mods many people dislike due to performance issues.

The main feature is the profit calculator, which calculates profit for dungeon chests. Good Mod has arguably the best profit calculator, for a variety of reasons:

  • A calculator determines whether or not kismetting a chest is worth based on expected value and current profit, and renders the GUI differently to inform the user of this;
  • The profit calculator determines whether using dungeon chest keys are worth, and renders a secondary square over the second best chest if it's worth;
  • The calculator is designed to work best in Croesus, but will still work inside individual chests, rendering in a similar style to SBE;
  • The API used to calculate profit is modular, meaning if one API goes down temporarily, the user can swap to another one. The API selection defaults to the Hypixel API to avoid putting unnecessary strain on independent developers' projects;
  • The user can manually select whether to use instasell or sell offer prices to compute profit (instabuy and buy offer, respectively, for chest keys), and these are selected per item as opposed to a universal toggle;
  • Renders a notification over chests/runs for which a kismet has already been used
  • Renders different colors in the main Croesus menu, indicating whether a chest has been opened. This rendering takes into account dungeon chest key logic.

There are a small number of other nice features, such as a notification for when Croesus is close to full, a running counter of how many chests are stored in Croesus, and a running loot counter for your lifetime dungeon loot.

Developer Information

Good Mod is written in Kotlin. Obligiatory shoutout to my good friend Odtheking, owner of Odin/OdinClient for porting my mod from Java to Kotlin. The funniest part is when I originally coded my mod, I intentionally named my variables very poorly; for example I called a config menu AwA and I called a font renderer object qualcommSnapDragon. Because funny or something, and also because I didn't expect to expand my mod as much as I did. He went through the entire thing and fixed (most of) the variable names, so thanks Od! Go check out his mod; it has both a cheat and a legit version, Odin and OdinClient, respectively.

I'm not a developer so don't expect any of the code to be good if you decide to look at it. I would prefer to query updated prices on checking each chest, but unfortunately this is just not feasable without creating my own massive server. Third party APIs don't in general update fast enough for there to be any change, and working directly with the Hypixel API is much more difficult, due to the sheer volume of information to be parsed. I run a protocol to update sell prices for dungeon items every 30 minutes, and when running this via the Hypixel API, it can easily take 20-40 seconds. Most of this is parsing the entire auction house.

To keep a running counter of Croesus chests, I need to access Hypixel's profile API endpoint on login, but this requires an API key. To get around this issue, I had to make my own server, [which i havent done yet finish this later]. I post the user's Minecraft UUID to my server because I think data is interesting.

After initially accessing the user's Croesus data, I store a local variable containing the Croesus information, and manually update it based on certain events. This also includes kismet data.
